# Java中的线程和多线程

## 基本概念
线程是操作系统能够进行运算调度的最小单位。它被包含在进程中，是进程中的实际运作单位。一条线程指的是进程中一个单一顺序控制流，一个进程中可以并发多个线程，每个线程并行执行不同的任务。  
线程是独立调度和分派的基本单位。  
统一进程中的多个线程将共享该进程中的所有资源，如虚拟地址空间，文件描述符和信号处理等。单统一进程中的多条线程有各自的调用栈，自己的寄存器环境，自己的线程本地存储。

## 线程的状态
在Java中的线程，具有物种基本状态:
- 新建 (New)
- 就绪 (Runnable)
- 运行 (Running))
- 阻塞 (Blocked)
- 死亡 (Dead)

### **状态流转图：**

#### 流转图说明
- 当 `Thread t = new Thread()`时，线程进入新建状态
- 调用 `t.start()` 方法，线程进入就绪状态，等待CPU调度执行，并不代表马上执行。
- 当CPU调度到就绪状态的线程时，线程运行，线程进入运行状态。 **就绪**状态是运行状态的唯一入口。
- 处于运行中的线程由于某种原因，放弃对CPU的使用权，线程进入阻塞状态，等到其再次进入就绪状态才有机会被CPU调度到运行状态。
- 当线程执行完或者因为异常退出了run()，线程结束。
